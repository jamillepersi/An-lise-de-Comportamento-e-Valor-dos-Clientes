# 📊 Análise de Comportamento e Valor dos Clientes (SQL + MySQL)

## 📌 Visão Geral do Projeto

Este projeto realiza uma análise exploratória de dados de clientes utilizando **SQL (MySQL)** com o objetivo de entender o comportamento de compra, identificar clientes de alto valor e gerar insights estratégicos para apoio à tomada de decisão.

A análise foca em responder perguntas de negócio reais como:

- Quem são os clientes mais valiosos?
- Como se comporta a frequência de compras?
- Quais clientes estão inativos?
- Existe relação entre frequência e valor gasto?
- Quais mercados (cidades) geram mais receita?

---

## 🎯 Objetivo

Transformar dados transacionais em **insights de negócio acionáveis**, apoiando estratégias de:

- Fidelização de clientes
- Segmentação de público
- Reativação de clientes inativos
- Expansão de mercado
- Aumento de receita

---

## 🛠️ Tecnologias Utilizadas

- MySQL
- SQL (JOINs, GROUP BY, agregações, subqueries)
- Modelagem relacional básica
- Análise exploratória de dados (EDA)

---

## 🧠 Estrutura da Análise

O projeto foi dividido em análises estratégicas:

### 1. Receita por cliente
Identificação dos clientes que mais geram faturamento.

### 2. Frequência de compras
Clientes com maior recorrência de compra.

### 3. Última compra (recência)
Identificação de clientes ativos e inativos.

### 4. Visão completa do cliente (RFM simplificado)
Combinação de:
- Frequência
- Valor gasto
- Recência

### 5. Receita por cidade
Identificação dos mercados mais lucrativos.

### 6. Receita por gênero
Análise de comportamento por segmento demográfico.

### 7. Clientes inativos
Identificação de risco de churn.

### 8. Frequência x Valor gasto
Análise de comportamento de compra vs geração de receita.

### 9. Top 10 clientes
Ranking de clientes estratégicos para fidelização.

---

## 📊 Principais Insights

- Clientes mais frequentes nem sempre são os que mais geram receita.
- Existe forte potencial de segmentação entre clientes de alto valor e alta recorrência.
- A recência de compra é um indicador crítico de risco de churn.
- Algumas cidades concentram maior parte da receita, indicando oportunidades de expansão regional.
- Um pequeno grupo de clientes representa grande parte do faturamento (princípio de Pareto).

---

## 📌 Conclusão

Este projeto demonstra como SQL pode ser usado para transformar dados brutos em insights estratégicos de negócio.

A análise permite apoiar decisões em marketing, retenção de clientes e crescimento de receita através de uma abordagem orientada a dados.
